# Reef — Security Review

**Scope:** all contracts in `src/` — identity/vault/index/bonds (AgentIdentity, AgentVault, AgentIndex, ReputationBond, Seasons, SignalMarket), the trust/risk layer (TrustOracle, TrustOracleConsumer, ReefGuard, ReefGuarded, Allocator, MockProtocol), the rate store (MethRate), and the adapters.
**Type:** internal review + a static-analysis & adversarial pass (Slither 0.11.5 + manual review), **not a third-party audit.** Reef is hackathon/testnet code; the Sepolia instances are demo-only and the one mainnet position (the mETH vault) is **demo scale and deposit-paused** — Reef **must not hold real mainnet TVL until the open items below are resolved and an external audit is completed.**

## Adversarial review pass (2026-06-14, multi-agent)

Three parallel adversarial agents (scoring path / capital path / off-chain + cross-layer) hunted for **novel** gameable or value-extraction vectors beyond #1–#20; every finding was re-verified against source by the reviewer before any action. Result: findings #21–#28 above. The two live-relevant integrity gaps (#24, #25, plus #26) are **fixed live** (the leaderboard now shows the authoritative on-chain Trust Score, and the off-chain recomputation reproduces `scoreOf` exactly). The contract findings that *are* cleanly fixable (#21 reputation-writer binding, #23 vault eviction, #27 bond cooldown) are **fixed in source with regression tests** (full suite 225 pass / 1 skip) but not redeployed — none is exploitable on the current single-operator deployment, and redeploying the root `AgentIdentity` is a full re-wire deferred to the next deploy. The subtle accounting findings (#22 withdraw-ratchet, #28 spot-mark share pricing) are **documented and audit-deferred**: both require a deployed/DEX-marked strategy that the live vaults don't have, and correct fixes touch core share/PnL math best done under audit.

## Fixed/mitigated in the security-audit pass (2026-06-12)

Ran Slither 0.11.5 (no new exploitable findings beyond the known CEI event-ordering and informational timestamp comparisons) plus two adversarial manual reviews of the session's new contracts. Each finding was re-verified against the source before acting.

- **#16 (live mitigation).** The live Mantle-mainnet mETH `AgentVault` (`0x76f129…cFA5`) was **paused** by its guardian — `deposit` is `whenNotPaused`, so no second (victim) depositor can enter; `withdraw` is never gated, so the sole depositor (the operator) can always exit. This closes the inter-depositor path on the immutable live instance. The proper multi-depositor fix is the realized-PnL / unit-consistent redesign (`totalUnderlying` in asset units, ETH-mark surfaced as a display-only view), deferred to the pre-mainnet audit with #13/#15.
- **#17 step cap.** `MethRate.setRate` now requires each push to move the stored rate by ≤ `MAX_STEP_BPS` (5%) of its current value — an honest keeper push (a few bps per ~8h) is unaffected, but a compromised key/RPC can no longer jump the rate in one transaction. (Bounds, doesn't eliminate; the durable fix is dedicated-key custody + realized-PnL reputation.)
- **#20 fixed.** `TrustOracle.registerVault` now requires the vault to be bound to the oracle's own `AgentIdentity` (`identity() == address(identity)`), and a new `removeVault(uint256 agentId)` lets the governor drop a broken vault *without calling it* (clears `vaultOf`/`isRegistered` + swap-pops the array), so one reverting cohort member can't permanently brick the public reads. Tested (`test_registerVault_rejectsWrongIdentity`, `test_removeVault_dropsFromCohort_andAllowsReRegister`).
- **ReputationBond reentrancy guard (defense-in-depth).** Slither flagged the fund-moving paths (`postBond`/`withdrawBond`/`openDispute`/`resolveDispute`/`claimExpiredStake`). They are already checks-effects-interactions correct (state updated before every `safeTransfer`) and the asset tokens in use have no transfer hooks, so it was not exploitable — but the contract was the one fund-custodying contract without an explicit guard. It now inherits `ReentrancyGuard` and marks those five entry points `nonReentrant`, closing the latent exposure if a callback-style asset token were ever used. Bond unit + invariant suites still green.

## Donation-proof reputation redesign (#15, drawdown leg of #19)

The headline manipulation — a bare token **donation** into a vault ratcheting `highWaterNav` and minting ERC-8004 reputation with no real yield (#15), which the live testnet's mintable mock made free, and which the `Seed` script + test helpers literally used as a shortcut — is now closed at the source:

- **`AgentVault.managedIdle`** is internally-accounted idle: it only moves on `deposit`/`recall` (in) and `deploy`/`withdraw` (out), so a raw `transfer` into the vault is **not** counted. New view **`reputableNav() = (managedIdle + strategy mark) * 1e18 / totalShares`**.
- **`publishReceipt` now credits reputation off `reputableNav()`** (high-water still applies). A donation lifts the public `nav()` (share pricing, where the virtual-offset #2 already neutralizes it) but **not** `reputableNav()` — so it mints zero reputation. Regression-locked: `test_publishReceipt_donationDoesNotCreditReputation`.
- **`TrustOracle` and `Allocator` drawdown components read `reputableNav()`** too, so an agent can no longer mask a real loss with a donation to keep its public Trust Score high (the donation leg of #19).
- **Share pricing is deliberately unchanged** (`nav`/`totalAssets` still use raw balances), so the solvency/invariant/fork suites are unaffected. The `Seed` script and the test reputation helpers were updated to create reputation via **real strategy yield** (realized on a strategy adapter) instead of a donation — same values, donation-proof path. Full suite green (**219 pass / 1 skip**).

**Realized-recall extension (#13).** A second pass took the redesign all the way to realized PnL: the strategy portion of `reputableNav` is now valued at **`deployedCostBasis`** (what was deployed, tracked on `deployToStrategy`), **not** the adapter's live `totalUnderlying()` mark. So an inflated/flash-loaned spot mark moves reputation by **zero** — only a `recall` that returns MORE than cost (the surplus lands in `managedIdle`) lifts `reputableNav` and credits reputation, and a closed position writes off any residual cost as a realized loss. This retires #13 at the source. Regression-locked: `test_reputation_creditsRealizedRecallNotMark` (an unrealized 2× mark credits 0; the realized recall credits the real delta). Share-pricing is still untouched — invariants/solvency/fork suites green.

**Live on the Sepolia leaderboard (2026-06-14).** #13/#15 are no longer source-only: the live testnet leaderboard was **redeployed onto the hardened contracts** so the displayed Trust Score is donation-proof in practice, not just in the repo. The prior live instance ran the old `nav()`-based vaults (`reputableNav()` reverts on them); the new instance — fresh `AgentIdentity` `0xDc9a22…6Be6`, 5 `reputableNav` vaults, `TrustOracle` `0x8085Db…F5E4` (reads `reputableNav`, Mantlescan-verified), wired Allocator/Guard/Seasons + re-registered in canonical ERC-8004 — is what `reef.gudman.xyz` now serves. The off-chain `trust_score.py` was switched `nav()`→`reputableNav()` to match. On-chain `scoreOf` ↔ off-chain `scores.json` parity ≈0.1%. The old (gameable) instance is abandoned, not upgraded (contracts are immutable). Freshness/cohort/centralization below still apply.

**#19 cohort-relative — addressed (configurable).** `TrustOracle` and `Allocator` now carry a governor-set **`reputationTarget`**: `0` (default) keeps the cohort-relative basis (`rep / cohort-max`, a within-set ranking); a non-zero value switches the reputation component to an **absolute** scale (`min(rep / reputationTarget, 1)`), so "best of a weak field" no longer reads as top trust. Default-off keeps existing behavior + parity with `trust_score.py`; a deployer that exposes `scoreOf` to external capital can opt into absolute. Tested: `test_setReputationTarget_absoluteScale`.

**Still genuinely open (honest):**
- **#19 freshness** — a no-op receipt still refreshes `lastReceiptAt`. This is **inherent**: freshness measures recency of a (cheap) signed heartbeat, so it can't be made un-fakeable without conflating it with the reputation/performance signal. A `minReceiptPeriod` would only rate-limit spam (and would break the live 10-min receipt cadence), so we document the semantics — freshness is a liveness signal, not proof of performance — rather than bolt on a cosmetic gate.
- **#16 mETH unit mismatch** — share-math, **mitigated by the deposit-pause**. The unit-consistent fix (token-denominated `totalUnderlying` + an ETH mark as a display-only view) would *undo* the mainnet vault's "NAV reflects real yield" demonstration and only changes immutable-irrelevant source (the live vault can't be upgraded), so it's a product decision held for audit rather than applied unilaterally.
- **#22 withdraw-ratchet** — **not live-exploitable** (live vaults hold no deployed strategy; the seed fully recalls). Held for audit, not patched in a rush, because the root cause is that `managedIdle` does double duty (real-idle tracker *and* half the reputation numerator) and `deployToStrategy` does an unguarded `managedIdle -= amount` — so a naive withdraw-side change risks an underflow there. **Audit-ready remediation:** decouple reputation accounting from `managedIdle` — track realized PnL in a dedicated accumulator that only moves on `recallFromStrategy` (realized gain/loss), and derive `highWaterNav`/reputation from that, so deposit/withdraw (which only move shares + cash) can never ratchet reputation. Regression test to add: a fuzz invariant that `reputableNav` is non-increasing across any `withdraw`.
- **#28 spot-mark share pricing** — **not live-exploitable** (the live adapter's mark is time-deterministic, not third-party-movable; the mETH vault is paused). Held for audit because the fix touches the **solvency-critical** `nav()`/`totalAssets()` path. **Audit-ready remediation:** for adapters whose `totalUnderlying()` is a manipulable spot quote (DEX-marked), price deposit/withdraw shares off a manipulation-resistant basis — `deployedCostBasis` or a TWAP of the mark — rather than the live spot, and/or forbid deposit+withdraw in the same block as a mark-moving op. Keep the spot mark only for display NAV.
- **Centralization → multisig (pre-mainnet commitment).** The live governor/arbiter/keeper roles are single EOAs (throwaway testnet keys). The transfer paths exist in-code (`transferArbiter`/`acceptArbiter`, `transferGovernor`, `setGovernor`, `transferKeeper`). **Commitment:** before any mainnet deployment, all governance roles migrate to a **2-of-3 Gnosis Safe** (fault-tolerant to one lost/compromised key). Deferred on testnet because wiring a multisig to throwaway keys protects nothing real and a mistaken single-step `setGovernor` could brick a live role.

## Fixed in this pass

- **#1 Reputation access control (vault-only).** `giveFeedback` is now gated to `reputationSource[agentId]`, which the agent's own wallet designates via `setReputationSource` (default-closed — no source set means no one can write). The intended source is the agent's `AgentVault`. `SignalMarket` no longer credits reputation, which also structurally kills the free self-dealing reputation farm. Residual: an operator's *own* authorized vault can still over-report via `publishReceipt` — closed by **#4** below.
- **#5 Reentrancy + CEI.** Added a minimal `ReentrancyGuard` (`src/utils/ReentrancyGuard.sol`) and applied `nonReentrant` to `AgentVault.{deposit,withdraw,deployToStrategy,recallFromStrategy}`, `AgentIndex.{deposit,withdraw,rebalance}`, and `SignalMarket.purchaseSignal`. Reordered `AgentVault.withdraw` to burn shares **before** the external adapter `recall` (checks-effects-interactions).
- **#8 SignalMarket.** `createListing` now requires `priceWei > 0`; `purchaseSignal` rejects `providerAgentId == consumerAgentId` (blocks zero-cost self-dealing reputation farming) and is `nonReentrant`. (Full mitigation also depends on #1.)
- **#4 NAV-derived reputation.** `publishReceipt` credits the vault's real on-chain per-share NAV delta (`nav()` change since the last receipt); the operator's claimed `navDelta` is ignored. Verified: a claimed `1e24` credits only the real `5e17`.
- **#3 Protocol adapter allowlist.** New `src/AdapterRegistry.sol` (governor-controlled). `AgentVault.approveStrategy` now requires the adapter to be registry-approved, so an operator cannot point a vault at an adapter that lies about `totalUnderlying()` to inflate NAV (hence reputation and index weight). This is a second key on top of the operator's own approval. **Allowlisting is by adapter ADDRESS, not codehash** as originally framed: Solidity embeds immutable variables into runtime bytecode, so instances of the same adapter type have distinct `EXTCODEHASH`es — a codehash allowlist would reject legitimate instances. The governor reviews each deployed instance; the testnet-only `MockYieldAdapter` (mints freely) must never be approved on a registry that gates real TVL.
- **#2 First-deposit / donation inflation.** `deposit`/`withdraw` in `AgentVault` and `AgentIndex` now use a `+1` virtual shares/assets offset. This removes the empty-vault 1-wei→1-share edge and makes any price-inflating donation a net loss to the attacker rather than a theft from the next depositor. First real deposit still mints 1:1.
- **#6/#10 ReputationBond.** `openDispute` now rejects the agent's own operator (no self-slash farming/griefing) and enforces **one active dispute per agent**, so a depleting bond can never owe more upheld slashes than it holds (deterministic, order-independent payouts). The single-arbiter centralization (#10) is now addressed in code: the arbiter is rotatable via a 2-step `transferArbiter`/`acceptArbiter` handoff, so it can be moved to a multisig/timelock after deploy.
- **#7 SafeERC20.** New minimal `src/utils/SafeTransferLib.sol` (treats "call succeeded and returned empty OR true" as success). Every external asset `transfer`/`transferFrom`/`approve` in `AgentVault`, `AgentIndex`, `ReputationBond`, and the adapters routes through it, so Reef supports USDT-style tokens that return no bool (proven by `test/mocks/NoReturnERC20.sol`).

## Fixed in the adapter-review pass

- **#11 Realized-vs-marked withdrawal.** `AgentVault.withdraw` priced the payout off `totalAssets()` (which includes a strategy's *spot mark*) but then ignored `recall`'s return value and transferred the full marked amount. A mark-to-market adapter (e.g. a DEX position) can realize slightly less than its mark when it actually sells, and the `IStrategyAdapter` contract explicitly lets `recall` return less than asked (a drained/slashed adapter, or rounding on a real AMM curve). Withdraw now pays `idle + recalled` — the amount the strategy actually delivered — so it can never overdraw the vault (reverting the last withdrawer) or pay one user out of another's principal. Shares are burned before the recall, so the withdrawer bears their own realization slippage and remaining holders stay whole. Regression-tested with an adapter that under-delivers (`test_withdraw_paysRealizedNotMarked_whenRecallUnderdelivers`). Severity is Med→Low in practice: because `totalUnderlying` quotes via `getAmountsOut` (fee included), a full drain realizes the mark exactly at realistic sizes, so the gap is a sub-wei rounding corner — but the fix removes the edge entirely.
- **#12 Sandwichable forced sale.** `FusionXAdapter.recall` swapped with `amountOutMin = 0`, so a forced withdrawal sale could be sandwiched down to an arbitrary price. It now bounds the sale with the same `maxSlippageBps` tolerance as `deploy`.
- **#14 Index realized-vs-marked withdrawal.** `AgentIndex.withdraw` had the same flaw as #11 one layer up: it priced the redemption off `totalAssets()` (which includes each vault's `nav()` mark) but transferred the full marked amount, ignoring what `_pullFromVaults` actually realized from the vaults. A vault's recall can return less than its mark (a mark-to-market adapter), so the index could overdraw — reverting the last redeemer or paying one holder out of another's principal. Withdraw now burns shares first (CEI) and pays the realized balance (`min(assets, available)`). Regression-tested with a listed vault whose adapter under-delivers (`test_withdraw_paysRealizedNotMarked_whenVaultUnderdelivers`, reverts without the fix).

## Safety primitives (added in the Phase 4 decentralization pass)

- **Circuit breaker** — `src/utils/Pausable.sol`. A guardian (the operator for an `AgentVault`, the governor for `AgentIndex`) can pause the risk-taking entry points (`deposit`, `rebalance`, `deployToStrategy`). **Withdrawals are deliberately never gated**, so a pause halts new exposure without ever trapping user funds. The guardian is rotatable.
- **withdrawPool reserve** — `AgentIndex.reserveBps` (governor-set). `rebalance` allocates only `total − reserve` to vaults, keeping an idle liquidity buffer in the index so redemptions are serviceable without recalling from vaults.
